# İkincil Pazar Trade

Hans birincil pazardan 120 token aldı (95€). 60. günde nakde ihtiyacı var, 97.5€'dan satıyor. Erkin alıyor.

## Aktörler

- **Hans** (mevcut sahip, satıcı)
- **Erkin** (yeni alıcı)
- **Backend** + **OrderBook.sol**

## Akış

```
1. Hans /portfolio → token 42'yi seç → "İkincil pazarda sat"
2. Hans 97.5€ × 120 token ask emri açar
3. Frontend → InvoiceNFT.approve(orderBook, tokenIds)
4. Frontend → POST /api/orders { type: 'ask', secondary: true }
5. Order book güncellenir:
     - Birincil ask hâlâ var (Ahmet 95€ × 180 kalan)
     - İkincil ask (Hans 97.5€ × 120) eklendi
6. Erkin ekrana bakıyor — best ask hâlâ 95€ (Ahmet'in birincil ask'ı)
7. Sara (başka yatırımcı) Ahmet'in 95€ ask'ını yer (180 token kalanı)
8. Şimdi best ask 97.5€ — Hans'ın askı
9. Erkin 97.5€'dan 60 token alır
10. Settlement → 60 token Hans'tan Erkin'e
11. Hans 60 × 97.5 = 5.850 USDC alır (komisyon kesintili)
12. Hans'ın realized P&L: 60 × (97.5 - 95) = 150 USDC kar (60 günde %2.6)
```

## Birincil ile İkincil Tek Bir Order Book

`OrderBook.sol` her iki tipte de aynı kontrat. Tek fark seller adresi:
- Birincil: `seller = KOBİ`
- İkincil: `seller = mevcut token sahibi`

Kontrat seller'ın token'a sahip olduğunu zaten check ediyor (ERC-1155 balance).

## Komisyon Farkı

| Trade tipi | Platform fee | Sigorta payı |
|---|---|---|
| Birincil | %1.0 | %0.5 |
| İkincil | %0.25 | %0 |

İkincil işlemlerde platform daha düşük fee alır çünkü sigorta zaten ilk satışta finanse edilmiş. Yatırımcı-yatırımcı transferinde yeni risk yok.

## Time-Decay Etkisi

Vade yaklaştıkça token nominal'e (100€) yakınsar:

```
Gün 0:    95.0€  (5% iskonto, 120 gün vade)
Gün 30:   95.8€  (4.2% iskonto, 90 gün vade)
Gün 60:   97.5€  (2.5% iskonto, 60 gün vade)
Gün 90:   98.8€  (1.2% iskonto, 30 gün vade)
Gün 120:  100.0€ (vade — Lidl ödüyor)
```

Bu monoton artış teorik; pratikte alıcı default sinyali fiyatı düşürebilir.

Detay: [Time-Decay Game Theory](/docs/game-theory/secondary-liquidity)

## Edge Cases

- **Token sahipliği değişti ama emir hâlâ aktif** → Cancel zorunlu
- **Vade çok yaklaşmış (24 saat kala)** → İkincil pazar otomatik kapanır
- **Default sinyali geldi** → Tüm askler iptal, sadece "force sell" mümkün
