# Fatura Onboarding

KOBİ'nin fatura yüklemesi → GİB doğrulama → NFT mint → marketplace'e listeleme.

## Aktörler

- **KOBİ** (Ahmet Bey)
- **Backend** (`/api/invoices`, GİB mock)
- **InvoiceNFT.sol**

## Adım Adım Akış

```
1. KOBİ /upload sayfasına gider
2. Fatura PDF + metadata girer (alıcı, tutar, vade, ülke)
3. Frontend → POST /api/invoices/draft
4. Backend → GİB mock → MFKS check (çift finansman var mı?)
5. Backend → Buyer rating al
6. Backend → KOBİ'ye fee tahmini döndür
7. KOBİ token sayısı + ask price belirler
8. Frontend → Wallet imza:
     mintInvoice(metadata, totalTokens) [InvoiceNFT.sol]
9. Contract Transfer event yayınlar
10. Backend event listener → Invoice doc'u "minted" yapar
11. KOBİ otomatik /invoice/[id]'ye yönlendirilir
12. KOBİ "Listele" butonuna basar
13. Frontend → placeAsk(invoiceId, price, qty) [OrderBook.sol]
14. Marketplace'te görünür
```

## Validasyonlar

| Katman | Ne kontrol ediyor |
|---|---|
| Frontend (zod) | Şema, gerekli alanlar, tarih formatı |
| Backend | KOBİ KYC durumu, alıcı whitelist, rating |
| GİB Mock | Fatura no benzersiz, MFKS conflict yok |
| Smart Contract | Caller authorized, totalTokens > 0, dueDate > now |

## Hata Senaryoları

- **GİB doğrulama başarısız:** Frontend toast + retry
- **Alıcı whitelist'te yok:** "Alıcı eklenmek için inceleme bekliyor"
- **Wallet imzayı reddetti:** Draft kaydedilir, sonra retry
- **Mint reverted:** Backend draft'ı "failed" işaretler

## Veri Modeli
