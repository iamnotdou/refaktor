# Sigorta Talep Akışı

Default ilan edildiğinde token sahipleri **claim yapmadan** otomatik tazminat alır. Pasif claim modeli.

## Aktörler

- **InsurancePool.sol** (claim ödeyici)
- **Escrow.sol** (default'u ilan eden)
- **Token sahipleri** (alıcılar)

## Pasif vs Aktif Claim

| Model | Avantaj | Dezavantaj |
|---|---|---|
| **Pasif** (otomatik dağıtım) | Kullanıcı eylem gerekmiyor, gas ödemiyor | Tek tx'te N transferi yapma maliyetli |
| **Aktif** (kullanıcı claim çağırıyor) | Gas maliyeti kullanıcıda, daha basit kontrat | Kullanıcı claim yapmazsa para kilitli |

MVP için **pasif claim** + token sahibi sayısı 50'den fazlaysa **aktif** fallback kullanılır.

## Pasif Akış

```
1. escrow.declareDefault(invoiceId)
2. escrow → insurancePool.notifyDefault(invoiceId, totalAmount)
3. InsurancePool:
   - getTokenHolders(invoiceId) [event log scan]
   - foreach holder:
       transfer USDC pro-rata
4. Tüm transferler tamamlanınca event: ClaimsDistributed
```

## Aktif Akış (Fallback)

Eğer holder sayısı çok fazlaysa (50+):
```
1. escrow.declareDefault(invoiceId)
2. InsurancePool.markClaimable(invoiceId, totalAmount)
3. Her holder kendi başına çağırır:
     insurancePool.claim(invoiceId)
4. Holder claim yapmadan 90 gün geçerse, kalan miktar geri pool'a dönüş
```

## Hesaplama

```
Default amount: 42.500 USDC (token sahiplerinin payı)
Insurance pool: 100.000 USDC mevcut
Token holders: Hans (60), Erkin (60), Sara (200), Mehmet (80), Diğer (100)
                                                            Toplam 500 token

Hans      → 60/500  × 42.500 = 5.100 USDC
Erkin     → 60/500  × 42.500 = 5.100 USDC
Sara      → 200/500 × 42.500 = 17.000 USDC
Mehmet    → 80/500  × 42.500 = 6.800 USDC
Diğer     → 100/500 × 42.500 = 8.500 USDC
Insurance pool yeni bakiye: 100.000 - 42.500 = 57.500 USDC
```

## Stake'çilerin Kaybı

Sigorta havuzu kullanıldığında stake'çilerin **toplam payı azalır**:
```
Önce: 100k pool, 100 stake'çi → 1.000 USDC/stake'çi
Sonra: 57.5k pool, 100 stake'çi → 575 USDC/stake'çi
Stake'çilerin kaybı: -42.5%
```

Bu yüzden stake'çiler default oranını yakından izler ve:
- Yüksek default ortamında unstake eder
- Düşük default + yüksek volüm ortamında stake artırır

Detay: [Insurance Incentives Game](/docs/game-theory/insurance-incentives)

## Subrogation (Halefiyet)

Sigorta tazminatı ödendikten sonra Lidl'e karşı **alacak hakkı** sigorta havuzuna devredilir. Lidl sonra ödeme yaparsa:
```
Lidl 90 gün sonra ödeme yaptı (geç): 50.000 USDC
- Yatırımcılar zaten sigorta'dan 42.500 aldı
- Geri kalan 7.500 + sigorta'nın subrogation alacağı 42.500 = 50.000
- Hepsi InsurancePool'a gider
- Pool yeni bakiye: 57.5k + 50k = 107.5k (önceki seviyenin üstüne çıkar)
```

Bu mekanizma sigorta havuzunu uzun vadede sağlam tutar.
