# Teknoloji Yığını

| Katman | Teknoloji | Neden |
|---|---|---|
| **Blockchain** | Polygon Amoy (testnet) | Düşük gas, EVM uyumlu, Türkiye'de aşina |
| **Smart Contract** | Solidity 0.8.x + Hardhat | En çok kaynak, hackathon dostu |
| **Frontend** | React 19 + Vite | Hızlı dev server, modern toolchain |
| **Wallet** | wagmi v2 + viem + RainbowKit | wagmi v2 stabilize, viem ethers.js'den hafif |
| **UI** | shadcn/ui + Tailwind v4 | Hackathon'da en hızlı UI çıkartma yolu |
| **State** | TanStack Query + Zustand | Server state TQ, client state Zustand |
| **Backend** | Node.js + Express + TypeScript | Standart, herkes biliyor |
| **DB** | MongoDB (Atlas free tier) | Şemasız flex, hızlı setup |
| **Realtime** | Socket.io | Order book updates, trade notifications |
| **Stablecoin** | MockUSDC (testnet) | Demo için, mainnet'te USDC |
| **Docs** | Fumadocs (Next.js + MDX) | Statik build, iyi DX |
| **Testing (contract)** | Hardhat + Chai | Standart |
| **Testing (FE)** | Vitest + Testing Library | Vite ile uyumlu |

## Karar Notları

`★ Insight ─────────────────────────────────────`
- **ERC-1155 vs ERC-721:** Faturayı 500'e bölmek için 1155 daha iyi; her token'a metadata gerek yok, hepsi aynı invoice.
- **Polygon yerine Arbitrum?** Hackathon'da Polygon Amoy faucet daha hızlı, dökümantasyon Türkçe daha çok. Phase 2'de multi-chain.
- **Off-chain matching neden?** On-chain matching her cancel transaction. LP'ler bunu ödemez, likidite kuruyacak. dYdX Serum modeli.
`─────────────────────────────────────────────────`
