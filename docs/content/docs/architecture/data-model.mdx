# Veri Modeli

## On-chain (Solidity Storage)

| Veri | Kontrat | Field |
|---|---|---|
| Token sahipliği | `InvoiceNFT` | `_balances[tokenId][address]` |
| Fatura metadata | `InvoiceNFT` | `invoices[tokenId] → Invoice struct` |
| Aktif emirler | `OrderBook` | `orders[orderId] → Order struct` |
| USDC kilitleri | `Escrow` | `locked[invoiceId]` |
| Sigorta stake | `InsurancePool` | `stakes[address] → StakePosition` |
| Trade event log | OrderBook (event) | `Trade` event |

## Veri Sahipliği Tablosu

| Veri | On-chain | MongoDB | Niye |
|---|---|---|---|
| Token sahipliği | ✅ Source of truth | Cache only | Decentralized invariant |
| Aktif emirler | ✅ Sadece settlement edilenler | ✅ Source of truth | Off-chain matching speed |
| Fatura metadata (özet) | ✅ Hash + key fields | ✅ Tam detay | On-chain pahalı, off-chain rich |
| KYC bilgisi | ❌ | ✅ Encrypted | KVKK / GDPR — kişisel veri off-chain |
| Trade history | ✅ Event log | ✅ Indexed cache | On-chain immutable, query off-chain |
| User profili | ❌ | ✅ | Off-chain only |
| Sigorta havuzu bakiye | ✅ Source of truth | Cache | On-chain |

## Tutarlılık Stratejisi

```
On-chain ←── source of truth ──> Off-chain (cache)
   │                                  │
   │                                  │
   └────── reconciliation job ────────┘
              (her 5 dk)
```

`backend/src/jobs/reconcile.ts` her 5 dakikada bir on-chain state ile MongoDB'yi karşılaştırır:
```typescript
async function reconcile() {
  const lastBlock = await getLastIndexedBlock();
  const currentBlock = await provider.getBlockNumber();
  
  const events = await contract.queryFilter('Trade', lastBlock, currentBlock);
  for (const ev of events) {
    await Trade.updateOne(
      { txHash: ev.transactionHash },
      { confirmed: true, blockNumber: ev.blockNumber },
      { upsert: true }
    );
  }
  
  await setLastIndexedBlock(currentBlock);
}
```
