# KOBİ (Satıcı)

Sistemin **birincil arz tarafı**. Faturayı yaratan, tokenize eden ve ask price belirleyen aktör.

## Profil

- **Tipik örnek:** Bursa tekstil, İzmir gıda, Gaziantep makina ihracatçısı
- **Ciro:** 5M TL – 200M TL/yıl
- **Tipik fatura:** 10.000€ – 500.000€, 60-180 gün vade
- **Web3 deneyimi:** Çoğunlukla **sıfır** — kripto cüzdanı bilmiyor

## Motivasyon

- Vade beklemek yerine **anında nakit**
- Banka kredisinden **daha ucuz** finansman
- Geleneksel faktoringe göre **daha düşük komisyon**
- Stoklarda kalan parayı tekrar üretime sokma

## Sistem İçindeki Görevleri

1. Wallet bağla (MetaMask veya gömülü wallet)
2. KYC tamamla (vergi no, imza sirküleri, ticaret sicil)
3. Faturayı yükle (PDF / e-Fatura XML)
4. GİB doğrulama bekle
5. Token sayısı ve ask price belirle
6. Listele
7. Kısmi satışlardan gelen USDC'yi takip et
8. Vade dolduğunda sigorta veya geç ödeme süreçlerini izle

## Ekranlar

- `/onboarding` — Wallet + KYC
- `/dashboard` — Aktif faturalar, satış durumu
- `/upload` — Yeni fatura yükleme
- `/invoice/[id]` — Spesifik fatura detayı, order book
- `/wallet` — Gelen USDC, withdrawal

## Smart Contract Etkileşimi

| İşlem | Contract | Fonksiyon |
|---|---|---|
| Fatura mint | `InvoiceNFT.sol` | `mintInvoice(metadata, totalTokens)` |
| Listeleme | `OrderBook.sol` | `placeAsk(invoiceId, price, qty)` |
| Listeden çekme | `OrderBook.sol` | `cancelOrder(orderId)` |
| USDC çekme | `Escrow.sol` | `withdraw(amount)` |

## KPI'lar

- **Time-to-cash:** Listeleme → ilk satış (hedef: 5 dk)
- **Fill rate:** Listelenen tokenın yüzde kaçı satıldı
- **Effective discount:** KOBİ'nin gerçek elde ettiği oran
- **Repeat rate:** Aynı KOBİ'nin tekrar listeleme oranı

## UX Riskleri

- Kripto cüzdan kurulumu engel — **gömülü wallet** veya **email login** gerekli (Phase 2)
- Fiat on-ramp olmazsa USDC'yi TL'ye çevirmek ek aşama
- GİB API hatalarında manuel onay süreci tasarımı şart
