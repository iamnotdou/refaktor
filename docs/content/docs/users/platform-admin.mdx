# Platform Admin

FaturaChain ekibinden, sistemin **operasyonel** tarafını yöneten persona. MVP'de manuel yönetim, Phase 3'te DAO'ya geçiş hedefli.

## Profil

- **Tipik örnek:** Platform kurucu ekibi, ileride DAO temsilcileri
- **Erişim:** Admin paneli, multisig wallet
- **Sorumluluk:** Listeleme onayı, alıcı whitelist, parametre güncelleme

## Görevleri

1. KOBİ KYC onaylama (manuel review queue)
2. Yeni alıcı (Lidl, Zara vb.) **whitelist** ve rating atama
3. Sigorta havuzu parametreleri (komisyon oranı, lock periodları)
4. Default durum yönetimi — geç ödeme uyarıları
5. Disputed fatura çözümü
6. Platform fee oranı güncelleme (multisig + timelock)

## Ekranlar

- `/admin/dashboard` — Sistem health, aktif faturalar, default rate
- `/admin/kyc-queue` — Bekleyen KYC'ler
- `/admin/buyers` — Whitelist, rating yönetimi
- `/admin/parameters` — Komisyon, sigorta oranı (multisig propose)
- `/admin/disputes` — İhtilaflı faturalar

## Smart Contract Etkileşimi

| İşlem | Yetki |
|---|---|
| Buyer whitelist | Multisig (3/5 imza) |
| Fee güncelleme | Multisig + 48 saat timelock |
| Pause emergency | Multisig (2/5 imza, panic switch) |
| Insurance pool params | Multisig |

## Yetki Modeli (MVP)

```
Owner (multisig) → InvoiceNFT, OrderBook, Escrow, InsurancePool
  ├── pause() / unpause()
  ├── setFee()
  ├── whitelistBuyer()
  └── upgradeImplementation() [proxy ile]
```

## DAO'ya Geçiş Hedefi

Phase 3'te **token holder'lar** yönetimi devralır:
- Buyer whitelist → DAO oylaması
- Fee güncelleme → DAO oylaması + timelock
- Sigorta havuzu parametreleri → DAO

## KPI'lar

- **KYC turn-around** — başvuru → onay süresi
- **Buyer coverage** — whitelist'teki alıcı sayısı
- **Default response time** — geç ödeme tespit → aksiyon
- **Parameter change frequency** — yıllık parametre güncellemesi
