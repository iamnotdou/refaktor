# Kredi Skorlama

Buyer'ın (alıcı firmanın) ödeme olasılığını **A/B/C/D** rating'e indirgeme.

## Rating Tanımları

| Rating | Default rate | Örnekler | Komisyon payı (Phase 2) |
|---|---|---|---|
| **A** | %0.5 altında | Lidl, Bosch, Zara, Carrefour | 0.3% |
| **B** | 0.5-2% | Auchan, Tesco, mid-tier corps | 0.5% |
| **C** | 2-5% | Smaller corps, emerging market | 1.0% |
| **D** | >5% | KABUL EDİLMEZ | — |

## Phase 2 — AI Modeli

Girdiler:
- Şirket büyüklüğü (revenue)
- Sektör
- Ülke risk skoru
- Geçmiş ödeme verisi
- Public bond rating (S&P, Moody's varsa)
- Sosyal sinyaller (haberlerde negatif?)

Çıktı: 0-100 skor → A/B/C/D

```python
# Pseudo
score = 0.3 * size_score + 0.2 * country + 0.2 * sector + 0.3 * payment_history
rating = bucket(score, [85, 70, 50])  # A/B/C/D thresholds
```

## Skor Güncelleme

- **Statik:** Whitelist'e alınırken atanır, değişmez (MVP)
- **Dinamik:** Her default → bir rating düşer; her başarılı 100 ödeme → bir rating yükselir (Phase 2)

## Yatırımcı Açısından Kullanım

Marketplace filtresi:
```
[ ] A only
[ ] A + B
[X] All ratings
Min APY: ___
```

UI'da rating renkli badge:
- A → yeşil
- B → mavi
- C → sarı
- D → kırmızı (görünmez)

## Ülke Risk Modu

Buyer'ın bulunduğu ülkenin sovereign risk'i de etkiler:
- Almanya/Hollanda → 0 ek risk
- Türkiye → +1 not (Türk alıcı + Türk satıcı, currency risk yok ama transfer riski var)
- Argentina/Türkiye dış → +2 not

## Veri Kaynakları (Phase 2)

- Bloomberg API → public corp data
- D&B (Dun & Bradstreet) → SME credit
- GİB API → fatura geçmişi
- Web scraping → haber, sosyal medya

## Tartışma — Centralized vs On-chain

Buyer rating **off-chain** mi, **on-chain** mı?

**Off-chain:**
- Hızlı update
- AI/ML kullanılabilir
- Veri kaynakları sınırsız

**On-chain:**
- Şeffaf, manipülasyon zor
- Auditor güvenilir

MVP off-chain (basit), Phase 2 hibrit:
- AI off-chain hesaplar
- Hash on-chain'e yazılır
- Auditor cross-check yapabilir
