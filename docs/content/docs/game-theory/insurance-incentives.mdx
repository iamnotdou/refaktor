# Sigorta Havuzu Teşvik Oyunu

Sigorta havuzu üç oyunculu bir mekanizma:
- **Stake'çi** — sermaye yatırıyor, getiri alıyor, default'ta kaybediyor
- **KOBİ** — havuz büyükse iskontolu fiyatlar (yatırımcılar daha rahat)
- **Yatırımcı** — havuz büyükse risk azalır, ödemeye razı olduğu fiyat artar

## Stake'çinin Beklenen Getirisi

```
E[r_stake] = komisyon_geliri / TVL  -  default_ödemeleri / TVL
```

Pozitif olması için:

```
komisyon_geliri > default_ödemeleri
```

Yani **komisyon oranı default oranını karşılamalı**:

```
0.5% × volume  >  default_rate × volume
0.5%           >  default_rate
```

Yani default rate %0.5 altındaysa stake'çi karda. Üstündeyse zararda.

## Optimal Komisyon Tasarımı

Komisyon çok düşükse → havuz büyümez → default'a karşı yetersiz
Komisyon çok yüksekse → KOBİ pahalı bulur, geleneksel faktoringe döner

**Trade-off:**
- Düşük default rate (örn. A rating): %0.3 komisyon yeter
- Yüksek default rate (örn. C rating): %1.5 komisyon gerekir

Phase 2'de **risk-based pricing**:
```
Buyer rating A → 0.3% sigorta payı
Buyer rating B → 0.5%
Buyer rating C → 1.0%
Buyer rating D → kabul edilmiyor
```

## Moral Hazard Riski

`★ Insight ─────────────────────────────────────`
Sigorta havuzu varsa **KOBİ ve yatırımcı dikkatlerini gevşetebilir** — "nasılsa sigorta öder". Bu klasik moral hazard. Korunma: 
- KOBİ'nin overcollateral payı (her default'ta KOBİ önce kaybeder)
- Yatırımcının "deductible" benzeri eşik
- Sigorta tam ödeme değil, %95 kapsam
`─────────────────────────────────────────────────`

## Stake'çinin Game-Theoretic Davranışı

### Senaryo 1: Düşük Default Rate (örn. %0.2)
- Aylık APY: 0.3% × volume / TVL ≈ 12-15% yıllık
- Stake'çi memnun, daha çok yatırır
- TVL büyür → sigorta kapasitesi artar → yeni KOBİ'lere kapı açılır

### Senaryo 2: Yüksek Default Rate (örn. %2)
- Aylık APY: -1% (zarar)
- Stake'çiler unstake → TVL düşer
- Sigorta kapasitesi düşer → platform yeni faturayı kabul edemez
- **Bank run** benzeri spiral

### Senaryo 3: Catastrophic Default (örn. %10)
- Bir gün toplu default → havuz boşalır
- Stake'çiler tamamen kaybeder
- Sistem güvenilirliği yok olur

## Bank Run Önleme

Geleneksel sigortada düzenleyici (BDDK) bunu yapar. FaturaChain Phase 2'de:

1. **Lock period** — stake en az 30 gün kilitli, ani çekim yok
2. **Tier'lı APY** — uzun lock daha yüksek getiri (180g lock = +%5)
3. **Reinsurance** — sigorta'nın sigortası (Phase 3)
4. **Emergency cap** — günlük maksimum unstake limiti

## Yatırımcı Açısından Sigorta Premium'u

Yatırımcı sigorta varsa daha yüksek fiyat öder. Kuantitatif:

```
Δp = V × ΔP(recovery)
```

Sigorta yokken: `P(recovery) = 0.85` (overcollateral'den)
Sigorta varken: `P(recovery) = 0.99` (sigorta + overcollateral)

```
Δp = 100 × 0.14 = 14€ (token başına)
```

Yani yatırımcı 14€ daha fazla ödemeye razı (95€ yerine 95.7€). Bu fark **KOBİ'nin lehine** — sigorta KOBİ'ye iyi fiyat verir.

## Eşitlik Noktası

Stake'çi getirisi = yatırımcının ekstra ödediği premium
- Stake'çi `r` APY istiyor → yıllık getiri = `r × TVL`
- Yatırımcı premium = `Δp × volume`
- Denge: `r × TVL = 0.5% × volume`
- `TVL / volume = 0.5% / r`

`r = 10%` ise `TVL` yıllık volume'un %5'i kadar olmalı. `r = 20%` ise %2.5'i.
