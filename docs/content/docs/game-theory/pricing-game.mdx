# Fiyat Keşfi Oyunu

## Oyun Tanımı

**Oyuncular:** KOBİ (`S`) ve N adet yatırımcı (`I_1, ..., I_N`)

**Strateji uzayı:**
- KOBİ: Ask price `p_S ∈ [0, V]` (V = nominal değer, ör. 100€)
- Yatırımcı: Bid price `p_I ∈ [0, V]` ve quantity `q_I`

**Kazanç (payoff):**

```
KOBİ:        π_S = (p_trade - C) × q_traded
Yatırımcı:   π_I = (V - p_trade) × q_I × P(ödeme) - r × t × q_I

P(ödeme) = alıcının ödeme olasılığı (rating'e bağlı)
r        = yatırımcının zaman tercihi (alternatif yatırımdan APY)
t        = vade
```

## Beklenen Denge Fiyatı

İdeal teorik denge:

```
p* = V × P(ödeme) × e^(-rt) - risk_premium
```

Örnek: Lidl (A rating), 120 gün vade, yatırımcı %15 yıllık APY beklentisi
- `V = 100`, `P = 0.98`, `t = 120/365`, `r = 0.15`
- `p* = 100 × 0.98 × e^(-0.15 × 0.329) = 100 × 0.98 × 0.952 = 93.3€`

## Açık Pazar vs Kapalı İhale

**Tek noktalı ihale (geleneksel faktoring):**
- KOBİ tek bir bankaya başvurur
- Banka monopol fiyat verir (örn. 80€)
- KOBİ'nin bilgi avantajı yok

**Açık pazar (FaturaChain):**
- Her yatırımcı bid verir, en iyi fiyat kazanır
- N artar → fiyat `p*`'a yakınsar (Bertrand competition)
- KOBİ artık fiyat alıcısı değil, fiyat keşfedicisi

## KOBİ'nin Optimal Ask Stratejisi

KOBİ ask'ı çok yüksek koyarsa kimse almaz; çok düşük koyarsa para masada kalır.

```
Ask çok yüksek (98€): 
  Hiçbir bid eşleşmez → satış yok → 0 kazanç
Ask doğru (95€):
  Bidler eşleşir → 95€ × 500 = 47.5k anlık nakit
Ask çok düşük (90€):
  Bidler hızla yer → 45k anlık nakit, 2.5k bedava bıraktı
```

**İteratif keşif:** KOBİ önce 96€ koyar, eşleşme olmazsa 95€'ya çeker. İkincil piyasa derinliği yoksa daha agresif iner.

`★ Insight ─────────────────────────────────────`
KOBİ'nin gerçek stratejisi tek seferde değil, **dinamik ask güncelleme** olmalı. Ama bunu yapmak için piyasa derinliği bilgisi şart — order book'un public olması KOBİ'ye bilgi avantajı verir.
`─────────────────────────────────────────────────`

## Yatırımcının Bid Stratejisi

Yatırımcı için ikinci-fiyat etkisi var: en iyi bid eşleştiği için, kendi rezerv fiyatını söylemek **dominant strateji** değildir.

**Senaryo 1 — Tek alıcı**
Hans tek başına bid versin. KOBİ ask 95€. Hans 95€'dan üstüne çıkmazsa eşleşir. Sonsuza kadar 95.01 verir → kar maksimum.

**Senaryo 2 — Çok alıcı**
3 yatırımcı bid yarışıyor. En iyi bid eşleşir. Bu yüzden yatırımcılar kendi rezerv fiyatlarına yakın bid verir (yoksa rakipleri yer).

Sonuç: **Çok alıcılı pazarda fiyat KOBİ'nin lehine, az alıcılıda yatırımcının lehine.**

## Likidite ve Fiyat Çarpıklığı

Düşük likiditede:
- Bid-ask spread geniş (95€ ask vs 92€ best bid → 3€ spread)
- KOBİ "para masada bırakmamak" için ask düşürür → suboptimal
- Çözüm: Likidite teşvikleri (LP rebate)

Yüksek likiditede:
- Spread daralır (95.5€ ask vs 95.3€ bid → 0.2€)
- Fiyat $p^*$'a yakın
- Tüm taraflar memnun

## Eşitsizlik (Information Asymmetry)

**KOBİ daha iyi biliyor:**
- Alıcının (Lidl) ödeme geçmişi
- İhracat sözleşmesinin detayları

**Yatırımcı daha iyi biliyor:**
- DeFi opportunity cost
- Likidite ihtiyacı (kendi)

Platform'un rolü asimetriyi azaltmak: kredi rating, GİB doğrulama, default rate raporları.

## Demo'da Görünür Olan

Demo'da Ahmet 95€ koyacak, Hans 95.8€ bid verecek (favorable slippage), eşleşme 95€'da olacak. Bunu jüriye:

> "Hans aslında 95.8€'ya kadar ödemeye razıydı. Ama açık pazar 95€'da eşleştirdi — yatırımcının lehine 0.8€ slippage. Bu **price-time priority** kuralı: maker (Ahmet) önce geldi, fiyatı o belirledi."

Sonra Mehmet'in 94€ bid'ini gösteririp: "Mehmet daha düşük teklif verdi, Ahmet kabul etmediği için bekliyor. Eğer Ahmet hızlı nakit isterse 94€'ya çekebilir."
