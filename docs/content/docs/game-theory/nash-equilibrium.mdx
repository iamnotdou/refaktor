# Nash Dengesi

Bir Nash equilibrium'da hiçbir oyuncu tek başına stratejisini değiştirerek kazancını artıramaz.

## 4 Aktörlü Sistem

| Aktör | Strateji değişkeni |
|---|---|
| KOBİ (`S`) | Ask price `p_S`, token sayısı `n` |
| Yatırımcı (`I`) | Bid price `p_I`, quantity `q_I` |
| Stake'çi (`K`) | Stake amount `s_K` |
| LP (`L`) | Spread `Δ_L`, depth `d_L` |

## Eşzamanlı Optimal Stratejiler

### KOBİ'nin Best Response

```
p_S* = V · E[P(ödeme)] · e^(-rt) - spread_L
```

LP spread'i daralırsa KOBİ daha iyi fiyat alır.

### Yatırımcı'nın Best Response

```
p_I* = p_S* + ε     (ε küçük, rakipleri geçmek için)
```

Rakip yatırımcı sayısı artarsa `ε` küçülür → fiyat `p_S`'a yakınsar.

### Stake'çi'nin Best Response

```
s_K* = s_max   eğer APY > r_alt
       0      aksi halde
```

Stake'çi APY > alternatif yatırım yieldı (örn. AAVE, Compound)'sa yatırır.

### LP'nin Best Response

```
Δ_L* = 2 · P(informed) · info_advantage
```

Bilgi asimetrisi azalırsa spread daralır.

## Sistem Dengesi

Tüm aktörler aynı anda best response oynarsa:

```
1. Bilgi asimetrisi düşük (rating, GİB, history)
2. → LP spread'i dar
3. → KOBİ iyi ask price alır
4. → Volüm yüksek
5. → Komisyon geliri yüksek
6. → Sigorta havuzu büyük
7. → Default tampon güçlü
8. → Yatırımcı risk premium düşük
9. → Yatırımcı yüksek bid verir
10. → KOBİ daha iyi fiyat alır (loop closes)
```

Bu **virtuous cycle** — pozitif geri besleme.

## Çoklu Denge Riski

Sistem birden çok dengeye sahip olabilir:
- **İyi denge:** Yüksek volüm, düşük spread, büyük havuz, %95 KOBİ memnuniyeti
- **Kötü denge:** Düşük volüm, geniş spread, küçük havuz, KOBİ'ler çekiliyor

İlk başta hangi denge gerçekleşeceği **beklentilere bağlı** (self-fulfilling).

## Kötü Denge'den Kurtulma

İlk dönemde platform sübvansiyon yapar:
- LP rebate
- KOBİ listing fee waiver
- Yatırımcı airdrop
- Stake'çi bonus APY

Bu sübvansiyonlar sistem kritik kütleyi yakaladıktan sonra kaldırılır → iyi dengeye kilitlenir.

## Mekanizma Tasarım Açısından

İyi mekanizma şunları sağlar:
1. **Truth-telling dominant strateji** — yatırımcı gerçek rezerv fiyatını söyler
2. **Individual rationality** — herkes katılımdan kazanır (>0 utility)
3. **Budget balance** — platform geliri >= maliyet
4. **Pareto efficiency** — hiçbir oyuncu başkasını incitmeden iyileşemez

FaturaChain'in açık pazarı bunların **çoğunu** sağlar. Eksik:
- Truth-telling kısmen — bid stratejik (rakip varlığına göre değişir)
- Pareto efficiency tam değil — adverse selection bazı KOBİ'leri eler

## Sonsuza Tekrarlanan Oyun

Tek seferlik oyunda strategic default cazip; tekrarlanan oyunda **cooperation Nash equilibrium**:
- Buyer ödemezse → tüm KOBİ'ler bilgilendirilir → buyer pazardan çıkar
- KOBİ sahte fatura kesse → KOBİ KYC iptal → KOBİ pazardan çıkar
- Yatırımcı manipüle ederse → wash trading detect → ban

**Folk theorem** geçerli — yeterince yüksek discount factor (gelecekteki getirileri yeterince değerli görme) ile tüm aktörler dürüst stratejide kalır.
