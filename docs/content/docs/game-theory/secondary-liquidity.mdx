# İkincil Likidite Oyunu

Birincil pazar tek seferlik (KOBİ → Yatırımcı 1). İkincil pazar **sürekli likidite** üretir (Yatırımcı 1 → Yatırımcı 2 → ... → Vade).

## Likidite Neden Önemli?

Yatırımcının "exit option" değeri:

```
V_liquid = V_hold_to_maturity + liquidity_premium
```

Likit token'a yatırımcı **daha yüksek fiyattan** girer (daha düşük iskonto):
- İlikit (sadece vade) → 92€'ya alır
- Likit (her an satılabilir) → 95€'ya alır

Bu farktan **KOBİ kazanır**. Likidite KOBİ'ye iyi fiyat verir.

## Likidite Sağlayıcıları (LP)

### Kim LP olur?
- Profesyonel market maker (algo trader)
- Sigorta havuzu fonu
- Token tutan ama anlık satmak istemeyen yatırımcı

### LP'nin gain'i
Bid-ask spread:
- Bid: 95.2€ (alır)
- Ask: 95.5€ (satar)
- Trade hacmi 1000 token → 300€ kar

### LP'nin risk'i
- **Adverse selection:** Bilgili trader sadece "iyi" tarafta yer (LP zarar eder)
- **Inventory risk:** Çok token tutarken default olursa
- **Funding risk:** Likidite çekme baskısı (banka run benzeri)

## Spread Modeli

Glosten-Milgrom modelinin uyarlaması:

```
spread = 2 × P(informed_trader) × info_advantage
```

Düşük informed trader oranı → spread daralır → likidite artar.

Platform stratejisi: bilgi asimetrisini azalt (rating, GİB, default raporu) → informed trader avantajı azalır → spread daralır → herkesin kazancı artar.

## Çeyrek Saatte Bir Likidite Şoku

İkincil pazarda büyük emir gelirse:
```
Hans 100 token sat (ask 97.5€)
Best bid: 96€ (Sara)
Hans pazarı düşürmek istemiyorsa:
  - Limit ask 97.5€ → bekle
  - Market sell 96€ → 1.5€ kayıp

LP'ler bu anda devreye girer:
  Bid 96.5€ × 100 — LP arbitraj fırsatını yakalar
  Hans 96.5€'da satar → 1€ kayıp (LP'siz 1.5€'ya göre iyi)
  LP 100 × (96.5 fill price → vade 100€) = 350€ kar
```

## Time-Decay Piyasası

Vade yaklaştıkça fiyat nominal'e yakınsar — bu **monoton** bir trend:
```
Gün 0:    p = 95€ (5% iskonto, 120 gün)
Gün 30:   p = 95.8€ (4.2% iskonto, 90 gün)
Gün 60:   p = 97.5€ (2.5% iskonto, 60 gün)
Gün 90:   p = 98.8€ (1.2% iskonto, 30 gün)
Gün 120:  p = 100€ (vade)
```

Bu **risk-free arbitraj** gibi görünür ama değil:
- Yatırımcının opportunity cost'u var
- Default riski sıfır değil
- İkincil piyasada exit zorluğu olabilir

## "Hold to Maturity" vs "Trade Out" Stratejisi

| Strateji | Ne zaman optimal |
|---|---|
| Hold to maturity | Yatırımcının likidite ihtiyacı düşük + default riski düşük |
| Trade out (50% gün) | Likidite ihtiyacı orta + ikincil pazar derin |
| Hızlı exit | Likidite şoku, default sinyali, daha iyi fırsat |

## İkincil Pazar Derinliği — Network Effect

Daha çok yatırımcı → daha çok ikincil işlem → daha derin order book → daha düşük spread → daha çok yatırımcı (positive feedback loop).

İlk yatırımcılar prematur — ikincil pazar boştur. Bu yüzden Phase 1'de **incentive program**:
- İlk 100 yatırımcıya komisyon iadesi (rebate)
- Volüm mil baş üstü → token airdrop (Phase 2)

## LP Rebate Mekanizması (Phase 2)

Order book'ta likidite koyan **maker** komisyon iadesi alır, alan **taker** komisyon öder:
```
Maker fee: -0.05% (rebate)
Taker fee: +0.30%
Net platform: +0.25%
```

Bu pattern **borsalarda standart** (Binance, Coinbase). LP'ler bid/ask koyar, taker'lar yer, LP rebate kazanır.
